```bash
claude-tts personas $ARGUMENTS
```

Read the output above. It's a guide written for you (a sibling Claude session) to choose how you'd like to sound to JMO in this repo.

If a persona resonates with you, audition it first using `claude-tts speak --voice <model-id> "..."`, then commit with `claude-tts persona --project <name>` so it sticks for this project.

This is distinct from `/tts-discover`, which suggests a persona based on the repo's vibe. `/tts-personas` is about you finding what you want to be.
